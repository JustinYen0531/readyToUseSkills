# Game Jam Design Canvas

Use this reference when you need the full structure behind the coaching flow.

## 00 Design Philosophy

- Core question: what should the player feel?
- Emotional cues: exploration, freedom, chaos, comfort, achievement, loneliness, healing, speed, horror, cooperation

## 01 Elevator Pitch

Formula:

This is a game about `[player identity]`, using `[special action or core mechanic]`, under `[setting or constraint]`, to `[reach the final goal]`.

## 02 Core Gameplay

List the three actions the player repeats most often.

## 03 Core Loop

Describe the repeating cycle in a small loop.

Format:

`[Action A]` -> `[Action B]` -> `[Action C]` -> `[Reward or reset to Action A]`

## 04 Hook

Define the one strongest selling point or eye-catching moment.

## 05 Controls

Map player input to actions.

## 06 Victory and Defeat

- victory condition
- defeat condition

## 07 MVP

Define what must work first:

- core mechanic
- one playable level or scenario
- one core system needed for the loop

Everything else is optional until the MVP works.

## 08 Assets

List only the art, animation, VFX, UI, and sound needed for the MVP.

## 09 AI Pipeline

Identify what non-programming work can be accelerated with AI:

- icons
- backgrounds
- music
- debugging assistance
- algorithm help

## 10 Tech

Specify:

- game engine
- collaboration stack

## 11 Team

Specify role ownership:

- design or integration
- programming
- art or animation
- AI-tool coordination

## 12 Timeline

Minimum milestones:

- Day 1 morning
- Day 1 afternoon
- Day 2 morning
- Day 2 afternoon
- final submission

## 13 Cut List

Rank which features to drop first when time runs out.

## 14 Demo Flow

Define what the player will see in:

- first 10 seconds
- around 30 seconds
- around 60 seconds

## 15 Final Checklist

Confirm:

- playable loop
- winnable and losable state
- no fatal bugs
- sound works
- UI is complete enough
- build exports successfully
